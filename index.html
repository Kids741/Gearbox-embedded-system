Gearbox Embedded Systems Curriculum

13-week journey through embedded systems, hardware prototyping, and access control projects.

Track each module, document progress, and build real-world solutions.

About Me Week 1 Journal

Module 1 · Introduction to Embedded Systems I

Weeks 1 – 3 · Documentation discipline, kit familiarization, and first microcontroller experiments.

1.1 Project Management & Documentation

  • Maintain ongoing academy progress updates.
  • Adopt Git-based version control practices.
  • Build a personal website highlighting the learning journey.

1.2 Introduction to Embedded Systems

  • Define embedded systems and key properties.
  • Explore kit components and their operating principles.

1.3 Distribution of Student Kits

Research every kit component to understand capabilities before deployment.

1.4 Overview of RP2040 & Raspberry Pi Pico

  • Study GPIO layouts and microcontroller architecture.
  • Prepare for software tooling with IDE overviews.
  • Week 3 focus: understand IDE roles, install Thonny, and learn Python basics.

1.5 – 1.8 First Firmware Experiments

  • Tour IDE options: Thonny, Arduino IDE, Pico C/C++ SDK.
  • Install Thonny, MicroPython, and supporting toolchains.
  • Blink the Pico LED using MicroPython and then C/C++.
  • Refine the blink routine with timing and behavior variations.

Module 2 · Introduction to Embedded Systems II

Weeks 4 – 6 · Circuit fundamentals, breadboarding, and sensing basics.

2.1 Circuit Literacy

  • Review basic electrical principles.
  • Assemble LED circuits on a breadboard.
  • Integrate push buttons for user input.

2.2 – 2.3 External Interfaces

  • Wire external LEDs, switches, and pushbuttons to the RP2040.
  • Incorporate resistors for protection, pull-up, and current-limiting roles.
  • Build LED-based projects such as multi-state traffic lights.

2.4 – 2.6 Sensors & Actuators

Week 6 project: Simple embedded system I.

  • Inputs: PIR sensor for motion, ultrasonic sensor for distance.
  • Outputs: LED 1 reacts to motion, LED 2 varies brightness with distance.
  • Deep dives on ultrasonic sensing principles and PIR behavior.

Module 3 · Introduction to Embedded Systems III

Weeks 7 – 8 · ESP32 onboarding and Arduino-based prototyping.

3.1 ESP32 & Arduino IDE

  • Understand the NodeMCU ESP-32S platform and update firmware.
  • Connect boards via Arduino IDE and review C/C++ fundamentals.
  • Write Arduino sketches, starting with blinking the onboard LED.

3.2 Multi-LED Builds

Create LED traffic control projects on the ESP32 to practice timing, states, and I/O management.

Module 4 · Microcontroller Programming & Access Control II

Weeks 9 – 10 · Communication protocols and multi-controller projects.

4.1 Protocol Fundamentals

  • Survey UART, I2C, SPI, and their use cases.
  • Compare advantages, disadvantages, and typical applications.

Capstone Project · Simple Embedded System II

  • Controller + AHT10 sensor + RFID reader.
  • Display sensor/RFID data on an OLED.
  • Send readings via serial to a secondary controller (ESP32 ⇄ Pico) and document both scenarios.
  • Deploy an ESP32-based web server to expose data online.

4.2 – 4.5 Cross-Controller Communication

  • Practice SPI, I2C, and UART links between Raspberry Pi Pico and ESP32.
  • Demonstrate bidirectional serial messaging.
  • Understand how the ESP32 connects to the internet and serves web pages.

Module 5 · Access Control Systems III

Week 11 · Relays, door locks, and system integration.

  • Interface the Raspberry Pi Pico with relays for high-voltage switching.
  • Research commercial electric door lock types, specs, pros, and cons.
  • Assignment: implement an end-to-end RFID access control system.
  • Complete Module 5 assessment focusing on access control architecture.

Module 6 · Student Project

Weeks 12 – 13 · Design and build a custom access control system leveraging the full kit.

  • Define problem statements and system requirements.
  • Prototype hardware, firmware, and documentation for a complete solution.
  • Deliver a working demo and capture lessons learned.
